Documentation follows these standards:

- **Tone**: Professional, institutional, no emojis
- **Structure**: Clear headings, logical flow, table of contents
- **Examples**: Code examples for technical concepts
- **Diagrams**: ASCII diagrams for architecture
- **Links**: Cross-references between documents
- **Audience**: Multiple audience levels (users, developers, ops)

To contribute:
1. Follow the style of existing documentation
2. Use clear, professional language
3. Include code examples where appropriate
4. Add diagrams for complex concepts
5. Link to related sections
